# ESP Widget

<p align="center">
  Настольный ESP32-S3 виджет с круглосуточной индикацией погоды, времени, автобусов, нагрузки ПК,
  музыки, громкости, локальных датчиков и удалённым управлением с Windows-приложения.

## О проекте

`ESP Widget` это связка из трёх частей:

- прошивка для устройства на `ESP32-S3`;
- desktop-приложение на `Python + Flet`;
- локальный backend, который связывает ПК и устройство по `HTTP`, `UDP discovery` и `WebSocket`.

Проект задуман как домашний/настольный информационный виджет: устройство показывает важные данные на экране и подсветке, а Windows-приложение позволяет всё это быстро настраивать без перепрошивки.

## Что умеет проект

### Устройство

- Показывает время, дату и основной домашний экран с погодой.
- Загружает погоду по координатам через `OpenWeatherMap`.
- Отображает влажность, температуру, давление и погодные иконки.
- Показывает расписание автобусов на сегодня и завтра.
- Умеет выводить нагрузку ПК: `CPU`, `GPU`, `RAM`.
- Показывает локальные данные с датчика `BME280`.
- Отображает название трека и исполнителя при смене музыки.
- Показывает оверлей громкости при изменении системного уровня звука.
- Поддерживает настраиваемую LED-подсветку и несколько режимов эффектов.
- Поддерживает пользовательские цвета отдельных элементов интерфейса.
- Принимает и воспроизводит GIF-анимации, конвертированные в `RGB565`.
- Поддерживает sleep mode по расписанию включения и выключения экрана.
- На первом запуске поднимает собственную Wi-Fi точку доступа для настройки.
- Обновляется по OTA.

### Desktop-приложение

Во `Flet`-интерфейсе уже есть отдельные вкладки для:

- главной панели со статусом соединения, прогрессом GIF и OTA;
- настройки цветов элементов экрана;
- загрузки GIF и удаления лишних кадров перед отправкой;
- настройки яркости экрана;
- настройки яркости и режима подсветки;
- общих настроек: Wi-Fi, погода, координаты, расписание, сетевые порты;
- factory reset;
- OTA-обновления устройства.

Дополнительно приложение:

- сохраняет настройки локально через backend;
- умеет включать автозапуск backend в Windows;
- может работать через собранный инсталлятор `Inno Setup`.

### Backend

Backend отвечает за синхронизацию между ПК и ESP:

- поднимает локальный HTTP API на `http://127.0.0.1:8787`;
- хранит настройки в `backend/storage/settings.json`;
- поднимает WebSocket-сервер для ESP;
- рассылает `UDP discovery`, чтобы устройство нашло IP компьютера;
- передаёт на устройство настройки, команды, цвета, GIF и OTA-команды;
- отслеживает музыку в Windows через `winsdk`;
- отслеживает громкость через `pycaw`;
- отправляет на устройство загрузку `CPU`, `GPU`, `RAM`;
- парсит расписание автобусов с `transport.mos.ru`;
- умеет автоматически подстраивать яркость экрана и подсветки по погоде через `Open-Meteo`.

## Основные функции по разделам

| Раздел | Что делает |
| --- | --- |
| Экран | Яркость, автояркость по погоде, время включения и выключения, цвет акцента |
| Подсветка | Яркость, цвет, режим LED, автояркость по погоде |
| Погода | Координаты, API key, период обновления, отображение состояния погоды |
| Автобусы | До 4 источников расписания, фильтрация по интервалу времени, кэш на сегодня и завтра |
| Музыка | Отправка текущего трека и автора на экран |
| Громкость | Отображение системной громкости на отдельном экране |
| Мониторинг ПК | Передача текущей нагрузки CPU/GPU/RAM |
| GIF | Конвертация в RGB565, изменение размера, удаление кадров, отправка чанками |
| OTA | Отправка URL или локального пути прошивки, запуск обновления на устройстве |
| Цвета UI | Тонкая настройка цветов отдельных элементов каждого экрана |
| Первый запуск | Wi-Fi AP-режим и web-страница первичной настройки |

## Архитектура

```text
Flet UI
   -> HTTP API (aiohttp)
      -> ESP service
         -> UDP discovery
         -> WebSocket
            -> ESP32-S3 firmware
```

Дополнительно backend получает данные из Windows и отправляет их на устройство:

- музыка;
- громкость;
- нагрузка ПК;
- сохранённые настройки;
- пользовательские цвета интерфейса.

## API backend

Основные endpoint'ы:

- `GET /api/status` — статус подключения ESP, состояние GIF и OTA;
- `GET /api/settings` — текущие настройки;
- `PUT /api/settings` — сохранить настройки и отправить diff на ESP;
- `POST /api/esp/command` — произвольная команда на устройство;
- `POST /api/esp/color` — изменить цвет конкретного элемента UI;
- `POST /api/esp/display` — настройки экрана;
- `POST /api/esp/backlight` — настройки подсветки;
- `POST /api/esp/gif` — передача GIF;
- `POST /api/esp/ota` — OTA-команда.

## Быстрый старт

### Desktop + backend

```bash
pip install -r requirements.txt
python backend/main.py
python ui/app.py
```

По умолчанию UI ожидает backend на `127.0.0.1:8787`.

### Firmware

```bash
cd firmware
pio run
```

Для загрузки на плату:

```bash
pio run -t upload
```

Если используется `LittleFS`, потребуется отдельно загрузить файловую систему:

```bash
pio run -t uploadfs
```

## Особенности проекта

- Desktop-часть ориентирована на `Windows`, потому что использует `winsdk`, `pycaw` и автозапуск через `Task Scheduler` / реестр.
- Прошивка и backend уже умеют работать с сохранёнными настройками и автоматически восстанавливать цвета интерфейса после переподключения ESP.
- В репозитории есть установщик `ESP Widget` версии `1.0.0` на базе `Inno Setup`.
